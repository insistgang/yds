# 正式演示视频分镜

## 视频口径

视频统一使用以下定位：LinkAble 是面向视障与行动不便人群的边缘智能无障碍辅助通行系统，当前阶段是校园最后 20 米可稳定演示的 P0 原型。

视频禁止出现以下表述：

- 已经是成熟导盲设备。
- 可以替代盲杖或导盲犬。
- 可以保证视障用户完全安全。
- 已经完成完整 P0 自训练模型，除非后续确实训练完成并有实验记录。

## 镜头细化

### 0:00-0:15 项目标题与痛点

- 画面内容：匿名校园通道远景，叠加标题“LinkAble：校园最后 20 米边缘智能无障碍辅助通行原型”。
- 旁白要点：无障碍通行中的盲道占用、台阶、坡道和临时障碍会影响视障与行动不便人群的最后一段通行体验。
- 需要展示的证据：真实路线环境，但不需要展示检测结果。
- 是否可使用第 3A 日志：否。
- 隐私注意事项：避开校名、楼名、路人正脸、车牌。

### 0:15-0:45 系统架构

- 画面内容：摄像头画面、Jetson、终端日志三栏或流程图。
- 旁白要点：系统在边缘端完成摄像头输入、YOLO 检测、P0 事件聚合、模板化中文提示和结构化日志记录。
- 需要展示的证据：P0 主链路和四类事件：`blind_road_occupied`、`stairs`、`ramp`、`road_obstacle`。
- 是否可使用第 3A 日志：可以截取事件日志格式。
- 隐私注意事项：不要把云端、Flutter 或地图说成主链路依赖。

### 0:45-1:30 road_obstacle 端到端实时演示

- 画面内容：镜头接近椅子、箱子或自行车等代理障碍，终端同步出现事件日志。
- 旁白要点：当前 `yolo11n.pt` + COCO 代理类已经验证路面障碍端到端链路。
- 需要展示的证据：`[EVENT] road_obstacle spoken -> 前方有障碍，请注意避让。`、`[STATS] fps=30.00` 附近日志。
- 是否可使用第 3A 日志：是，3A 已有 17 次 `road_obstacle` 和 10 次 `spoken`。
- 隐私注意事项：自备道具优先，不展示连续原始图像流存储。

### 1:30-2:30 四类 P0 场景素材

- 画面内容：依次展示路障、台阶、坡道或无障碍入口、盲道占用素材。
- 旁白要点：路障已完成端到端预演；台阶、坡道、盲道占用正在进入 P0 数据采集、标注和自训练模型阶段。
- 需要展示的证据：每类 3-5 个短镜头、标注框示例、类别定义。
- 是否可使用第 3A 日志：仅 road_obstacle 可使用；其他类别不可用 3A 夸大。
- 隐私注意事项：素材需要匿名化，避免路人和地点标识。

### 2:30-3:15 日志、cooldown、FPS、隐私保护

- 画面内容：终端日志滚动、统计摘要、隐私保护原则叠字。
- 旁白要点：事件不会在同一障碍上持续刷屏；后续 FPS 稳定接近 30；系统默认不保存连续原始视频。
- 需要展示的证据：`road_obstacle: 17`、`spoken: 10`、`cooldown: 7`、`WARN_COUNT=0`、`ERROR_COUNT=0`、最后 5 条 FPS 统计。
- 是否可使用第 3A 日志：是。
- 隐私注意事项：不要展示 API key、真实身份信息或未匿名地点。

### 3:15-4:00 数据采集与自训练模型计划

- 画面内容：P0 数据表、标注规范、训练计划时间线。
- 旁白要点：下一阶段只围绕四类 P0：盲道占用、台阶、坡道、路面障碍，不加入红绿灯、人行横道等 P1 类别。
- 需要展示的证据：数据量目标 400-600 张、类别定义、标注规则。
- 是否可使用第 3A 日志：否。
- 隐私注意事项：展示匿名样例或裁剪后的素材。

### 4:00-4:30 总结

- 画面内容：路线终点、系统流程、阶段状态表。
- 旁白要点：LinkAble 当前完成了边缘端实时链路预演，下一步是 P0 数据采集和自训练模型补齐。
- 需要展示的证据：第 3A 结论“已达到”、后续计划。
- 是否可使用第 3A 日志：是。
- 隐私注意事项：避免绝对安全承诺和产品成熟化表述。
