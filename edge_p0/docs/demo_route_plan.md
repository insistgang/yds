# 校园最后 20 米正式演示路线计划

## 演示定位

正式路线用于展示 LinkAble 作为边缘智能无障碍辅助通行原型，在校园最后 20 米场景中完成“看得见、能理解、能提示、能记录”的 P0 主链路。

当前 `yolo11n.pt` + COCO 代理类只保证 `road_obstacle` 端到端预演。`stairs`、`ramp`、`blind_road_occupied` 需要通过真实 P0 数据采集、标注和后续自训练模型补齐。正式答辩材料必须区分“已跑通端到端链路”和“待训练 P0 自定义模型”。

## 路线总览

路线建议从楼外约 20 米处开始，沿一条真实人行通道接近楼宇入口。途中依次布置或选择路面障碍、台阶、坡道或无障碍入口、盲道占用点，终点到达楼宇入口。

| 点位 | P0 类别 | 场景目标 | 预期提示 | 所需素材 |
|---|---|---|---|---|
| A. 起点：楼外约 20 米 | 无事件或背景负样本 | 展示校园最后 20 米路线起点和设备启动状态 | 无提示或仅展示日志待机 | 环境远景、设备特写、空路面负样本 |
| B. 路面障碍 | `road_obstacle` | 证明端到端链路已跑通 | 前方有障碍，请注意避让。 | 障碍物、检测框、事件日志、中文提示 |
| C. 台阶 | `stairs` | 采集台阶 P0 素材，后续用于模型训练与演示 | 前方有台阶，请减速。 | 台阶正面、斜侧、近中远距离素材 |
| D. 坡道或无障碍入口 | `ramp` | 采集坡道或可通行入口素材 | 前方有坡道。 | 坡道入口、坡面、扶手或边界素材 |
| E. 盲道占用 | `blind_road_occupied` | 采集盲道被占用场景素材 | 前方盲道被占用，请注意绕行。 | 盲道、占用物体、结构化事件记录 |
| F. 终点：楼宇入口 | 无事件或路线完成 | 展示到达目标点和演示闭环 | 无提示或总结提示 | 入口远景、设备日志、路线收尾画面 |

## 点位 A：起点，楼外约 20 米

| 项目 | 内容 |
|---|---|
| 点位名称 | 楼外约 20 米起点 |
| 场景目标 | 交代正式演示路线的真实空间和设备启动状态，采集无障碍通道负样本 |
| 预期 P0 类别 | 无事件，作为背景负样本 |
| 推荐道具或真实环境 | 人行通道、楼宇外侧、无遮挡道路 |
| 推荐镜头角度 | 第一视角略向下，保留前方 3-8 米路面；旁路机位拍设备和路线 |
| 预期中文提示 | 无提示，或只保留日志中待机/FPS 信息 |
| 需要拍摄的素材 | 路线起点远景、设备佩戴/手持方式、终端日志启动画面 |
| 隐私注意事项 | 避开人脸、车牌、校名牌匾和人员姓名标识 |
| 失败时的替代方案 | 改用楼外任意 20 米无障碍通道作为匿名路线 |

## 点位 B：路面障碍 road_obstacle

| 项目 | 内容 |
|---|---|
| 点位名称 | 路面障碍点 |
| 场景目标 | 展示当前已验证的端到端闭环：USB 摄像头 -> YOLO -> P0 事件 -> 中文提示 -> 日志 |
| 预期 P0 类别 | `road_obstacle` |
| 推荐道具或真实环境 | 椅子、箱子、书包、自行车、临时施工桶等 COCO 容易识别的代理障碍 |
| 推荐镜头角度 | 第一视角对准障碍物中心，障碍物占画面 15%-40%；同时保留路面上下文 |
| 预期中文提示 | 前方有障碍，请注意避让。左侧或右侧场景可触发左前方/右前方提示 |
| 需要拍摄的素材 | 检测框画面、`[EVENT] road_obstacle` 日志、中文提示、FPS 统计、cooldown 片段 |
| 隐私注意事项 | 不拍摄路人正脸；障碍物优先使用自备道具 |
| 失败时的替代方案 | 换成更容易被 COCO 识别的人、椅子、箱子或自行车；缩短距离并保持光照稳定 |

## 点位 C：台阶 stairs

| 项目 | 内容 |
|---|---|
| 点位名称 | 台阶点 |
| 场景目标 | 采集台阶 P0 数据并为后续自训练模型准备演示素材 |
| 预期 P0 类别 | `stairs` |
| 推荐道具或真实环境 | 楼宇入口台阶、室外短台阶、台阶边缘清晰区域 |
| 推荐镜头角度 | 正面、左斜、右斜各一组；近距离 1-2 米、中距离 3-5 米各一组 |
| 预期中文提示 | 前方有台阶，请减速。带距离时可为“前方 2 米有台阶，请减速。” |
| 需要拍摄的素材 | 台阶全貌、接近台阶过程、标注参考截图、后续模型验证素材 |
| 隐私注意事项 | 避免拍摄楼宇实名标识、人员脸部和敏感位置 |
| 失败时的替代方案 | 先以图片序列或短视频素材做离线推理验证，再回到实地路线 |

## 点位 D：坡道 ramp 或无障碍入口

| 项目 | 内容 |
|---|---|
| 点位名称 | 坡道或无障碍入口点 |
| 场景目标 | 采集坡道、缓坡、可通行入口的 P0 数据 |
| 预期 P0 类别 | `ramp` |
| 推荐道具或真实环境 | 楼宇无障碍坡道、坡道入口、带扶手的缓坡 |
| 推荐镜头角度 | 从通道正面接近；侧向补拍坡度和入口边界；保留可通行区域 |
| 预期中文提示 | 前方有坡道。若方向稳定，可为“前方有坡道，位于右前方。” |
| 需要拍摄的素材 | 坡道入口、坡道主体、路径上下文、可见边界和扶手 |
| 隐私注意事项 | 不把地点名称和楼宇单位信息作为视频重点 |
| 失败时的替代方案 | 选择更开阔、边界更明显的坡道；用路标遮挡或裁剪敏感信息 |

## 点位 E：盲道占用 blind_road_occupied

| 项目 | 内容 |
|---|---|
| 点位名称 | 盲道占用点 |
| 场景目标 | 采集盲道被物体占用的 P0 数据，展示项目面向无障碍通行的核心价值 |
| 预期 P0 类别 | `blind_road_occupied` |
| 推荐道具或真实环境 | 自备箱子、椅子、书包、共享单车停放到盲道上的可控模拟场景 |
| 推荐镜头角度 | 画面同时包含盲道纹理和占用物体；近中远距离各一组 |
| 预期中文提示 | 前方盲道被占用，请注意绕行。带方向时可提示建议绕行方向 |
| 需要拍摄的素材 | 占用物体、盲道纹理、事件日志、标注参考截图 |
| 隐私注意事项 | 不拍摄真实违规个人信息；占用场景优先用自备道具模拟并及时恢复现场 |
| 失败时的替代方案 | 改用室外盲道模型、临时铺设纹理板或图片/视频输入完成素材补齐 |

## 点位 F：终点，楼宇入口

| 项目 | 内容 |
|---|---|
| 点位名称 | 楼宇入口终点 |
| 场景目标 | 展示校园最后 20 米路线完成，收束系统能力和日志证据 |
| 预期 P0 类别 | 无事件或根据入口环境触发 `ramp` |
| 推荐道具或真实环境 | 匿名楼宇入口、可通行门口、无障碍入口 |
| 推荐镜头角度 | 第一视角进入入口前 3-5 米；旁路机位拍设备和日志 |
| 预期中文提示 | 无提示，或根据入口实际坡道输出坡道提示 |
| 需要拍摄的素材 | 路线收尾、事件统计、FPS 摘要、隐私说明画面 |
| 隐私注意事项 | 避免出现学校名称、门牌号、人员身份信息 |
| 失败时的替代方案 | 只保留入口近景和日志截图，不展示可识别地点信息 |

## 正式演示判断标准

正式演示前至少确认：

- `road_obstacle` 仍可实时触发事件和中文提示。
- FPS 后续稳定接近 30。
- cooldown 不刷屏。
- 没有摄像头、模型加载、事件管线崩溃。
- 不保存连续原始图像流。
- 视频和文档清楚标注：COCO 代理类是链路预演，P0 自训练模型仍需补齐。
