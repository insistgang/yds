# P0 现场采集 Checklist

## 采集前检查

- Jetson 电源稳定，电量或供电线材充足。
- HP w300 USB 摄像头连接正常，图像节点使用 `/dev/video0`。
- 执行 `v4l2-ctl -d /dev/video0 -c exposure_dynamic_framerate=0`，避免动态曝光帧率影响 30 FPS。
- 确认存储空间充足，优先采集离散图片，不保存连续视频。
- 确认采集目录为 `datasets/linkable_p0_raw/`。
- 检查镜头中是否可能出现清晰人脸、车牌、校名、姓名牌、门牌等敏感信息。
- 如出现敏感信息，样本先放入 `rejected_privacy/`，删除或打码后再进入训练集。

## A-F 路线点位采集任务

| 点位 | 采集目标 | 推荐 label | 采集说明 |
|---|---|---|---|
| A 起点 | 空通道、背景环境、设备起步状态 | `negative` | 采集无遮挡路面和正常通行背景 |
| B 路面障碍 | 临时障碍物占用通行路径 | `road_obstacle` | 使用箱子、椅子、背包、路锥、自行车等可控道具 |
| C 台阶 | 楼宇入口或通道台阶 | `stairs` | 覆盖正面、左斜、右斜和不同距离 |
| D 坡道 | 坡道或无障碍入口 | `ramp` | 保留坡面、入口边界和可通行区域 |
| E 盲道占用 | 盲道被物体占用 | `blind_road_occupied` | 画面同时包含盲道纹理和占用物 |
| F 终点 | 楼宇入口收尾、正常路面 | `negative` 或 `ramp` | 根据入口是否存在坡道选择 label |

## 每个点位建议拍摄

- 远景：5-8 米，保留路线上下文。
- 中景：3-5 米，目标清晰可见。
- 近景：1-2 米，目标边界清晰。
- 左侧角度：从左前方接近目标。
- 右侧角度：从右前方接近目标。
- 不同光照：晴天、阴天、阴影、背光、傍晚或室内外过渡。
- 正样本：目标真实存在且清晰。
- 负样本：相似环境但目标不存在，便于后续误报检查。

## 每类建议数量

| 类别 | 建议数量 |
|---|---:|
| `road_obstacle` | 100-200 张 |
| `stairs` | 80-150 张 |
| `ramp` | 60-120 张 |
| `blind_road_occupied` | 80-150 张 |
| `negative` | 按点位补充，不进入 YOLO 类别 |

第一阶段总目标为 400-600 张正样本。

## 隐私处理规则

- 不采集清晰人脸、车牌、校名、姓名牌、门牌等可识别信息。
- 不上传原始图片或视频。
- 不保存连续视频作为默认流程。
- 对含敏感信息的图片，先删除；确需保留时必须打码后再进入数据集。
- 采集现场优先使用自备道具模拟障碍，并在采集后恢复现场。

## 拍摄后整理流程

1. 运行 `python3 scripts/check_p0_dataset.py --root datasets/linkable_p0_raw` 查看数量。
2. 人工复查 `manifest.csv`，确认 label、路线点位和 session 是否正确。
3. 做隐私筛查，将问题样本移入 `rejected_privacy/` 或删除。
4. 只把通过隐私检查的样本进入标注流程。
5. 标注完成后再生成 YOLO 格式数据集，按 train/val/test = 70/20/10 划分。


## 演示视频合规要求

### 着装要求
- 正装或商务休闲
- 必须搭配深色长裤或长裙
- 避免荧光色、过于休闲的服饰

### 匿名要求
- 画面中不得出现学校名称、校徽
- 不得出现人脸（如有需打码）
- 不得出现车牌、门牌号等可识别信息
- 不得出现导师姓名、联系方式
