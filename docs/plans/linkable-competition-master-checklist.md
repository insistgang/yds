# LinkAble 比赛版总清单（对齐 PRD v2.2）

> 当前版本：对齐 PRD v2.2
> 项目定位：**基于边缘智能的无障碍辅助通行原型系统**
> 主线目标（P0）：**摄像头/图片/视频输入 -> YOLO 目标检测 -> 事件聚合与去抖 -> 模板化中文提示 -> presenter_female 本地语音播报 -> 结构化事件记录**
> 一句话记忆：**无人驾驶让车看懂路，LinkAble 让弱势行人听懂路、让管理者看见盲区。**
> P1（增强项，不阻塞提交）：云端大模型长文本报告、FastAPI 接收结构化事件、Flutter 展示页、红绿灯/人行横道识别
> P2（本期不做）：实时大模型对话、动态路线规划、志愿者系统、WebSocket 推送

**目标：** 在比赛提交前，完成一套可稳定演示的无障碍辅助通行原型系统，能在固定点位或巡检场景下识别盲道占用/台阶/坡道/路面障碍，并通过结构化事件记录支撑城市无障碍设施治理。

**适用范围：**

- 已采购 `Jetson Orin Nano SUPER 8GB` 套件
- 当前相机：`IMX219 77°`（CSI）或 USB 摄像头（fallback）
- 当前供电：套件自带 DC 适配器
- 当前目标为"可稳定演示的原型系统"，不是成熟产品或完整 App
- 三种演示形态：固定点位监测（推荐）、手持/推车式巡检（辅助）、胸前佩戴（仅展示）

## 2. 总体验收

### 2.1 最终一句话验收

- [ ] Jetson 设备能够在真实场景下识别前方障碍，并在 `1.5s` 左右自动播报中文提示

### 2.2 量化验收标准

**性能指标：**
- [ ] 推理帧率 `>=30 FPS`
- [ ] 单帧推理延迟 `<100 ms`
- [ ] 检测到播报响应 `<1.5 s`
- [ ] P0 场景识别成功率 `>=80%`

**稳定性指标：**
- [ ] 同一障碍不会短时间重复刷屏播报（音频冷却生效）
- [ ] 即使没有云端和手机，边缘端仍能独立演示（本地优先）
- [ ] 录制视频时结果稳定，不依赖手工改数据

**数据价值指标（AGENTS.md 3.5）：**
- [ ] 结构化事件记录（时间、位置、类型、置信度）
- [ ] 可量化、可追溯、可治理的数据输出
- [ ] 支撑校园/社区无障碍管理方的治理决策

### 2.3 三个核心演示场景（校园最后 20 米）

- [ ] **A 点位**：盲道占用 -> "前方盲道被占用，请注意绕行"
- [ ] **B 点位**：台阶识别 -> "前方有台阶，请减速"
- [ ] **C 点位**：坡道识别 -> "前方有坡道，位于右侧"
- [ ] **D 点位**：路面障碍 -> "前方有障碍，请注意避让"

> 注：A-D 为固定点位路线，演示路线长度约 20 米，覆盖 P0 四类检测场景。
